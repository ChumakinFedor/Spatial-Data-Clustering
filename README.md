# K-means Clustering (C)

**K-means Clustering (C)** — это консольный проект кластеризации на языке C99: генерация синтетических 2D-данных, запуск K-means «с нуля», экспорт предсказаний в CSV и визуальное сравнение истинных и предсказанных меток.

## Архитектура под капотом
Проект реализует полный цикл кластеризации без внешних ML-библиотек в C-части:
* **Память и структура данных:** точки и центроиды хранятся в плоских массивах `double` (layout: `data[i * n_features + j]`).
* **Динамическое чтение CSV:** в `main.c` используется стратегия расширения буфера через `realloc` (стартовая емкость `100`, затем удвоение), что защищает от потери данных на больших входах.
* **API-модель (Caller Allocates):** функция `kmeans_run(const KMeansConfig *cfg)` не делает `malloc/free`; вызывающая сторона заранее выделяет `data`, `labels`, `centroids`, а также передает указатель на `inertia`.
* **Устойчивая K-means-логика:** улучшенная инициализация центроидов (spread/farthest-first), assignment/update-циклы, обработка пустых кластеров через переинициализацию наиболее удаленной точки.
* **Метрика качества:** после сходимости вычисляется **Inertia (WCSS)** и печатается в stdout с точностью до `10` знаков после запятой.

## Основные возможности
* **Чтение CSV с пропуском заголовка:** приложение читает `data.csv` формата `x,y,label` (или `x,y,true_label`) и корректно пропускает строку заголовка.
* **Надежная обработка ошибок ввода:**
  * пустой файл,
  * битые/`malformed` строки,
  * ошибки открытия/записи файлов,
  * ошибки выделения памяти.
* **Прозрачный I/O-пайплайн:**
  * вход: `data.csv`,
  * выход: `results.csv` (`x,y,pred_label`).
* **Контроль качества кластеризации:** печать количества прочитанных точек (`Read N points`) и финальной `Inertia (WCSS)`.
* **Визуализация результата:** `plot_results.py` строит 2 графика — слева истинные метки, справа предсказанные; центроиды отмечаются красным маркером `X`.

## Генерация данных
Запустите Python-скрипт для создания тестового датасета:
```bash
python3 generate_data.py
```
### Параметры генерации (в `generate_data.py`):
* `n_samples=300` — количество точек
* `n_features=2` — размерность (2D для визуализации)
* `centers=3` — истинное число кластеров
* `cluster_std=0.8` — разброс точек внутри кластера
* `random_state=50` — фиксированный seed для воспроизводимости
* Результат: Файл `data.csv` с колонками `x,y,original_label`

### Для изменения входных данных требуется:
**1. Изменить `generate_data.py`**

**Измените следующие переменные:**
* `n_samples=300` — измените количество точек
* `centers=3` — измените количество кластеров
* `cluster_std=0.8` — немного поменяйте разброс точек внутри кластера
* `random_state=50` — измените на любое другое число для генирации нового seed'а

**2. Изменить `main.c`**

**Поменяйте количество кластеров в 93 строке `const int k = 3`**
**!!!ОБЯЗАТЕЛЬНО!!! `centers=3` и `k = 3` должны совпадать**
```bash
const int k = 3;
```

## Ограничения
* **`k` задается вручную:** в текущей версии `k=3` фиксируется в `main.c`.
* **Чувствительность к инициализации:** несмотря на улучшенную стратегию старта центроидов, K-means может сходиться к локальным минимумам.
* **In-Memory модель:** рабочие структуры живут в памяти процесса; между запусками сохраняются только файлы `data.csv` и `results.csv`.
* **Фиксация размерности:** текущая конфигурация валидирует `n_features == 2`.
